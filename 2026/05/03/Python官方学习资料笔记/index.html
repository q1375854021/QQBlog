一、简介

执行 python --version 命令，输出显示当前版本为 Python 3.10，该版本已略显过时。Python 3.14 带来了众多新特性，例如 free-threaded 版本取消了 GIL 限制，因此我们将通过 pyenv 来安装 Python 3.14，获得更好的性能和最新的语言特性。

1.1 pyenv 简介

pyenv 是一个 Python 版本管理工具，允许在同一系统上安装和切换多个 Python 版本。它的核心原理是通过修改环境变量来实现不同 Python 版本之间的切换，使得开发者可以根据项目需求灵活选择合适的 Python 版本。

二、安装 Python 3.14

2.1 安装依赖包

在安装 pyenv 之前，需要先安装编译所需的依赖库。这些库包含了 Python 编译过程中必需的工具和动态链接库。

2.2 使用 pyenv 安装 Python 3.14

安装完依赖后，使用 pyenv 安装 Python 3.14 的 free-threaded 版本。版本号中的 t 表示无 GIL（Global Interpreter Lock），即自由线程版本，这意味着 Python 3.14t 允许多线程真正的并行执行，不再受 GIL 限制。

2.3 设置本地 Python 版本

2.4 Python 与 Python3 的区别

在大多数 Linux 系统中，python 命令指向 Python 2.x，而 python3 指向 Python 3.x。新系统通常已将 python 链接到 python3。pip 与 pip3 的关系同理，推荐使用 python3 和 pip3 以确保使用正确版本的包管理器。

三、Python 快速入门

3.1 数值运算

运算符 说明 示例 结果
/ 浮点数除法 13/7 1.8571428571428572
// 整数除法（向下取整） 13//7 1
** 幂运算 5**2 25

3.2 字符串基础

Python 中单引号 ' ' 和双引号 " " 均可用于定义字符串，两者在功能上等价，开发者可根据喜好选择使用。转义字符 \n 表示换行，如 'First line. \n Second line.' 会输出为两行。print() 函数会正确解析转义字符并输出格式化结果，而原始字符串不会处理转义字符。

三引号 """ 用于定义多行字符串，常用于文档字符串或保留格式的文本输出。

3.3 字符串重复与拼接

* 运算符用于重复字符串，如 3 * 'un' + 'ium' 结果为 'unununium'。相邻的字符串字面量会自动拼接，这种特性在处理长文本时可以提高可读性。

3.4 字符串索引与切片

以 word = 'Python' 为例，字符串支持从前往后从 0 开始索引，也支持从后往前使用负数索引：

表达式 说明 结果
word[0] 获取第一个字符 'P'
word[-1] 获取最后一个字符 'n'
word[0:2] 切片获取前两个字符 'Py'
word[:2] 等同于 word[0:2] 'Py'
word[-2:] 获取最后两个字符 'on'
word[4:] 获取从索引4到末尾 'on'
word[:2] + word[2:] 等于原字符串 'Python'

注意：字符串是不可变（immutable）类型，无法通过索引修改其中的字符，任何”修改”操作实际上都会创建新的字符串对象。

3.5 获取字符串长度

len() 函数返回字符串的字符数量，也可以用于列表、元组等可迭代对象。

3.6 列表基础

列表是可变（mutable）数据类型，可以随时修改其中的元素，包括添加、删除、修改操作。列表中的元素可以是任意类型，甚至可以混合不同类型。

3.7 变量与引用

Python 中的变量使用 = 赋值时，并不复制数据，而是创建对当前对象的引用。可以将变量理解为贴在对象上的”标签”，两个变量可能指向同一个对象。

  • id() 函数可以打印对象的内存地址，帮助理解引用关系
  • [:] 切片操作会为列表分配新的内存空间，创建原列表的副本
  • 使用 is 可以判断两个变量是否指向同一个对象

3.8 多重赋值

Python 支持一行内对多个变量同时赋值，这种语法称为元组解包，常用于交换两个变量的值。

四、流程控制

4.1 用户输入

input() 函数用于获取用户输入，该函数始终返回字符串类型，即使用户输入的是数字也不例外。如需进行数值计算，需使用 int() 或 float() 进行类型转换。

4.2 条件语句

Python 使用 if-elif-else 结构进行多条件判断，其中 elif 是 “else if” 的缩写。条件判断从上往下执行，一旦某个条件满足就执行对应的代码块并跳过其余条件。

4.3 for 循环

Python 的 for 循环用于遍历可迭代对象（如列表、元组、字符串、字典等），每次循环从可迭代对象中取出一个元素。在 Python 中，for 循环的语法与其他语言的 foreach 类似。

4.4 range 函数

range() 函数的区间是左闭右开的，即包含起始值但不包含结束值，常用于需要指定循环次数的场景。

4.5 字典迭代

字典的底层原理是哈希表，通过键的哈希值快速定位值，因此表现为键值对结构。使用 .items() 方法可以同时获取键值对，这在遍历字典时非常有用。在遍历过程中删除元素时，需要创建字典副本或新字典来避免运行时错误（字典在迭代过程中不允许修改大小）。

4.6 match 语句

match 语句是 Python 3.10 引入的模式匹配功能，类似于其他语言中的 switch-case，但功能更强大。match 语句支持字面量匹配、元组匹配、类实例匹配以及带条件的 guard 匹配。

4.7 for 循环的 else 子句

for 循环的 else 子句是 Python 的独特语法，在循环正常结束（未执行 break）时执行。这常用于配合 break 语句判断是否找到了目标元素。

4.8 pass 语句

pass 语句什么都不做，用作占位符。当语法上需要一条语句但程序不需要做任何操作时使用，例如定义空类、空函数，或作为 if 语句的占位。

五、数据容器

5.1 容器类型对比

Python 的内置容器类型各有特点，选择合适的容器类型可以提高程序效率：

类型 说明 索引复杂度 插入复杂度 空间利用率
List 动态指针数组，支持增删改 O(1) O(n) 较高
Tuple 静态指针数组，创建后不可变 O(1) - 极高
Dict 哈希表，键值对存储 O(1) O(1) 平均 较高
Set 只有 Key 的哈希表，去重高效 O(1) O(1) 平均 较高

为什么 Python 没有链表？ 因为链表容易导致 Cache Miss（CPU 缓存未命中），而 Python 已有解释器开销，若底层大规模使用链表，速度将会极慢。使用动态数组（list）在大多数场景下性能更优。

5.4 元组

元组是不可变对象，一旦创建就不能修改，这使得元组比列表更轻量且可以作为字典的键。列表是可变对象，适合需要频繁修改的场景。只有一个元素的元组需要在末尾添加逗号来区分于普通括号表达式。

5.5 集合

集合中的元素是无序的、不重复的，非常适合用于去重和集合运算。创建空集合只能用 set()，不能用 {}（空 {} 创建的是字典）。

5.6 字典

字典是键值对的数据结构，键必须是可哈希的（不可变类型），值可以是任意对象。使用花括号或 dict() 创建字典。

5.7 列表推导式

列表推导式是简化循环创建的简洁语法，将 for 循环和元素创建合并为一行。多个 for 循环或包含 if 条件时，按从左到右的顺序执行。

5.8 布尔运算

and 和 or 是短路运算符，从左至右求值，一旦可以确定结果就停止求值。这不仅提高效率，还能实现一些常用模式。

六、函数

6.1 函数定义与调用

函数是组织代码的基本单元，可以提高代码的复用性和可读性。函数执行时的变量查找遵循 LEGB 规则：Local（局部）→ Enclosing（闭包）→ Global（全局）→ Built-in（内置）。

6.2 局部与全局变量

函数内部默认只能访问全局变量，但如果直接赋值会创建局部变量。使用 global 关键字可以声明操作全局变量，但这通常不是好的设计实践。

6.3 可变对象传参

Python 中所有对象都是引用传递，但区分可变（mutable）和不可变（immutable）类型很重要。不可变参数（int、string、tuple 等）在修改时会创建新空间；可变参数（list、set、dict 等）可以在原地修改，影响外部变量。

6.5 返回多个值

函数可以返回多个值，实际上返回的是元组，调用者可以使用元组解包接收。

6.6 默认参数

默认参数在函数定义时创建，并绑定到函数对象上。如果默认参数是可变对象（如列表、字典），会导致多次调用共享同一对象的问题。推荐使用 None 作为默认参数，在函数内部初始化。

6.7 关键字参数

使用参数名指定参数，可以不按顺序传递，这提高了代码的可读性和调用的灵活性。

6.8 可变参数

*args 接收可变位置参数（打包成元组），**keywords 接收可变关键字参数（打包成字典）。

6.9 参数解包

使用 * 解包列表或元组，使用 ** 解包字典。这是调用函数时”逆操作”。

6.10 lambda 表达式

lambda 用于创建小型匿名函数，适用于简单且易读的场景。lambda 函数只能包含一条表达式，其结果就是返回值。

6.11 文档字符串

文档字符串用于描述函数、类、模块的功能，可以通过 __doc__ 属性访问。规范的文档字符串应该包含参数说明、返回值说明等。

6.12 函数注解

函数注解提供参数和返回值的类型提示，是参考建议而非强制约束。注解不会影响函数执行，但可以被 IDE 和类型检查工具利用。

PEP8 提示：代码被阅读的频率远高于被编写的频率，良好的代码风格很重要。

七、字符串方法

7.1 常用方法一览

字符串提供了丰富的方法来进行查找、替换、分割、连接等操作：

f-string 是 Python 3.6 引入的字符串格式化方式，在字符串前加 f 或 F，在花括号 {} 内可以直接嵌入变量和表达式，可读性最好。

八、模块

8.1 模块基础

模块是包含 Python 定义和语句的文件，其文件名是模块名加后缀 .py。通过 __name__ 全局变量可以获取模块名，当模块被直接运行时值为 '__main__'。

Python 提供了多种导入模块的方式，适应不同的场景需求：

8.3 主模块与 __main__

当直接运行脚本时，__name__ 为 '__main__'；当作为模块导入时，__name__ 为模块名。这个特性常用于在模块中添加测试代码或命令行入口。

8.4 模块搜索路径

模块搜索按以下顺序进行，找到第一个匹配就停止：

  1. 内置模块（sys.builtin_modules_names），如 time、os 等 C 扩展
  2. sys.path 中的路径：
    • 脚本所在目录（如果是导入）
    • PYTHONPATH 环境变量指定的目录
    • Python 安装包自带库的目录
    • pip 安装的第三方库目录

8.5 编译缓存

Python 在 __pycache__ 目录中缓存编译版本（.pyc 文件），下次导入时直接使用缓存，加快加载速度。使用 -O 或 -OO 可以生成优化过的字节码，减小文件大小。

8.6 包结构

包是包含 __init__.py 文件的目录，用于将多个模块组织在一起。__init__.py 文件可以包含包的初始化代码，__all__ 列表定义了 from package import * 时导出的模块。
