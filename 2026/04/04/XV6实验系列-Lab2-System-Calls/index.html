1.2 问题解答

问题 (1)：Looking at the backtrace output, which function called syscall?

解答：

通过 bt（backtrace）命令查看调用栈：

从调用栈可以看出，usertrap 函数调用了 syscall。#2 显示为 0x00000000，这是因为 usertrap 是由中断触发的，没有上层函数调用它，因此栈帧中保存的是无效地址。

知识点：在 RISC-V 架构中，当发生 trap/exception 时，硬件会自动保存一些状态信息，但不会保存完整的调用栈。因此对于中断处理函数，其返回地址可能是未知的。

问题 (2)：What is the value of p->trapframe->a7 and what does that value represent?

解答：

通过 GDB 命令 p p->trapframe->a7 可以查看到值为 7。

参照 user/usys.S 文件，a7 寄存器存放的是系统调用号。系统调用号 7 对应的是 exec 系统调用。

可以通过 p p->name 查看当前进程名，结果显示为 initcode 进程，这正是执行 exec 系统调用的进程。

知识点：在 RISC-V 的调用约定中，a0-a7 是参数/返回值寄存器。XV6 利用 a7 传递系统调用号，a0-a5 传递系统调用参数，返回值通过 a0 返回给用户程序。

问题 (3)：What was the previous mode that the CPU was in?

解答：

使用 GDB 命令查看 sstatus 寄存器：

结果为 0x200000022。其中 bit 8（SPP 位）为 0。

参照 RISC-V Privileged 架构手册中关于 Sstatus 寄存器的描述：

  • SPP (Supervisor Previous Privilege) 位表示 trap 发生前的特权级
  • SPP = 0：trap 是从 User Mode（用户模式） 产生的
  • SPP = 1：trap 是从 Supervisor Mode（监管者模式） 产生的

因此，本次 trap 是从用户模式产生的。

  • zero 是 RISC-V 的零寄存器，其值永远为 0

问题 (5)：Why does the kernel crash?

解答：

内核崩溃的原因是访问了空指针（地址 0）导致 Kernel crash。

地址 0 不属于任何合法的地址空间。参照教材 Figure 3-3（XV6 虚拟地址空间布局），可以看出虚拟地址 0 根本没有被映射到任何物理内存，因此访问该地址会触发页错误（Page Fault）。

scause 寄存器的值可以印证这一点。scause 的值为 0xd（十进制 13），根据 RISC-V Privileged 架构手册：

scause 值 名称 描述
0x8 Instruction page fault 指令页错误
0x9 Load page fault 加载页错误
0xb Store page fault 存储页错误
0xd Load page fault 加载页错误

scause = 0xd 表示发生了 Load Page Fault，即加载数据时发生了页错误。

参考资料：RISC-V Privileged 手册关于 trap 寄存器的说明

2. System Call Tracing

2.1 中断环境初始化原理

要完成系统调用追踪功能，必须理解 XV6 的中断处理流程。以下是详细的中断初始化链路：

(1) ecall 指令与异常处理

ecall 指令会根据当前 CPU 特权模式跳转到不同的异常处理函数中。

异常处理函数的入口地址通过 stvec（S 模式）或 mtvec（M 模式）寄存器设置。

(2) medeleg 和 mideleg 寄存器

通常情况下，所有特权模式的中断和异常都会交给 M 模式处理。但这种方式效率较低，因此 RISC-V 支持异常下放机制：

  • medeleg（Machine Exception Delegation）：将特定异常号的异常委托给 S 模式处理
  • mideleg（Machine Interrupt Delegation）：将特定中断号的中断委托给 S 模式处理

因此，当前 U 模式的 ecall 会直接交给 S 模式处理。

(3) 启动流程

  1. start 函数设置 mepc 寄存器，并开启定时器中断
  2. main 函数执行 mret 后，PC 跳转到 mepc 指定的地址，进入 S 模式
  3. main 函数中调用 trapinithart，设置 stvec 指向 kernelvec
  4. 当再次触发 trap 时，会进入 kernelvec（定义在 kernelvec.S 中）

RISC-V Trap 相关寄存器

寄存器 作用
stvec S 模式 trap 向量基地址寄存器
sepc S 模式异常程序计数器，保存触发异常的指令地址
scause S 模式异常原因寄存器
stval S 模式异常值寄存器，保存异常相关的特定数据
sstatus S 模式状态寄存器，SPP 位表示 trap 前的特权级

系统调用 编号

内存管理关键概念

  • Page Size (PGSIZE): 4096 字节 (4KB)
  • 空闲链表: 空闲页通过链表连接，每个页头部 8 字节存储 next 指针
